# AI Bot 平台 - 项目目标文档

## 项目定位

一个面向 AI 应用岗位的 **简化版 Coze 平台**，支持用户通过**双模式**（画布可视化 + 表单配置）设计 AI 工作流和智能体，具备完整的 RAG 知识库能力和动态技能扩展机制。

---

## 核心目标

### 1. 双模式工作流编排（核心亮点）

用户可以通过**两种等价方式**创建工作流，且**可随时切换**：

#### 1.1 画布模式（可视化编排）
- React Flow 拖拽式节点编辑器
- 节点拖拽添加、连线连接
- 画布缩放、平移、网格对齐
- 节点选中后的配置面板
- 实时预览工作流结构

#### 1.2 表单模式（结构化配置）
- 表格式节点列表（类似问卷星）
- 每行代表一个节点，可排序
- 节点类型选择下拉框
- 节点参数表单配置
- 条件分支的表格化表达

#### 1.3 双模式转换
- 画布 ⟷ 表单 实时双向同步
- 画布操作自动更新表单数据
- 表单修改实时反映到画布
- 统一的 DSL JSON 作为中间格式
- 切换时状态保持不丢失

#### 1.4 支持的节点类型
| 节点 | 画布展示 | 表单配置 |
|------|---------|---------|
| 开始节点 | 圆形输入框 | 变量定义表单 |
| LLM 节点 | 方形AI图标 | 模型选择 + Prompt 编辑 |
| 知识库节点 | 书本图标 | 知识库选择 + 参数 |
| 条件分支 | 菱形判断框 | 条件表达式表格 |
| 代码节点 | 代码图标 | 代码编辑器 + 语言选择 |
| 工具/MCP节点 | 插件图标 | 技能选择 + 参数映射 |
| 结束节点 | 圆形输出框 | 输出变量定义 |

## 双模式工作流详细设计

### 画布模式界面
```
┌─────────────────────────────────────────────────────────────────┐
│  [画布]  [表单]  ← 模式切换 Tab                                  │
├──────────┬──────────────────────────────────────────┬───────────┤
│          │                                          │           │
│  节点库   │           画布区域                        │  配置面板  │
│  ─────   │         (React Flow)                     │           │
│  🟢 开始 │                                          │  节点类型  │
│  🔵 LLM  │    ┌────────┐      ┌────────┐           │  ───────  │
│  📚 知识 │    │  开始   │─────→│  LLM   │           │  模型选择 │
│  🔀 条件 │    └────────┘      └───┬────┘           │  Prompt  │
│  🔧 工具 │                        │                │  温度设置 │
│  🔴 结束 │                   ┌────┴────┐          │           │
│          │                   │  条件   │          │  [保存]   │
│          │                   └───┬──┬──┘          │           │
│          │                      ↓  ↓             │           │
└──────────┴──────────────────────────────────────────┴───────────┘
```

### 表单模式界面
```
┌─────────────────────────────────────────────────────────────────┐
│  [画布]  [表单]  ← 模式切换 Tab                                  │
├─────────────────────────────────────────────────────────────────┤
│  工作流名称: [________________]                                  │
│                                                                 │
│  ┌─────────┬──────────┬────────────────┬────────────────┐      │
│  │  顺序   │  节点类型 │    节点配置     │    条件/跳转    │      │
│  ├─────────┼──────────┼────────────────┼────────────────┤      │
│  │    1    │   开始   │  输入变量:query │      -         │      │
│  ├─────────┼──────────┼────────────────┼────────────────┤      │
│  │    2    │   LLM   │  模型:gpt-4    │      -         │      │
│  │         │          │  Prompt:...    │                │      │
│  ├─────────┼──────────┼────────────────┼────────────────┤      │
│  │    3    │  条件分支 │  条件:意图判断  │ 意图A→跳转5     │      │
│  │         │          │                │ 意图B→跳转6     │      │
│  ├─────────┼──────────┼────────────────┼────────────────┤      │
│  │   ...   │   ...    │      ...       │      ...       │      │
│  └─────────┴──────────┴────────────────┴────────────────┘      │
│                                                                 │
│  [+ 添加节点]  [导入JSON]  [导出JSON]  [保存工作流]              │
└─────────────────────────────────────────────────────────────────┘
```

### DSL JSON 示例（双模式共用）
```json
{
  "version": "1.0",
  "name": "智能客服工作流",
  "nodes": [
    {
      "id": "start_1",
      "type": "start",
      "position": {"x": 100, "y": 100},
      "data": {
        "outputs": ["user_query"]
      }
    },
    {
      "id": "llm_1", 
      "type": "llm",
      "position": {"x": 300, "y": 100},
      "data": {
        "model": "gpt-4",
        "temperature": 0.7,
        "prompt": "你是客服助手，用户问题：{{user_query}}"
      }
    },
    {
      "id": "condition_1",
      "type": "condition", 
      "position": {"x": 500, "y": 100},
      "data": {
        "conditions": [
          {"expr": "{{llm_1.intent}} == 'product'", "target": "knowledge_1"},
          {"expr": "{{llm_1.intent}} == 'human'", "target": "end_1"},
          {"expr": "default", "target": "llm_2"}
        ]
      }
    }
  ],
  "edges": [
    {"source": "start_1", "target": "llm_1"},
    {"source": "llm_1", "target": "condition_1"}
  ]
}
```

## 用户场景

### 场景1：智能客服（画布模式）
1. 点击"新建工作流"，选择"画布模式"
2. 从左侧拖拽节点：开始 → LLM → 条件分支 → 知识库/结束
3. 连线配置流程走向
4. 点击 LLM 节点，配置 Prompt 和模型参数
5. 保存并绑定到智能体

### 场景2：快速配置（表单模式）
1. 点击"新建工作流"，选择"表单模式"
2. 逐行添加节点，选择类型和配置参数
3. 在条件列设置分支逻辑
4. 点击"预览画布"查看图形化效果
5. 切换到画布模式微调布局

### 场景3：导入导出
1. 在画布模式设计好工作流
2. 导出为 JSON 文件
3. 在表单模式导入 JSON
4. 继续编辑后再次导出

## 技术栈规划

### 前端
- **Vue 3 + TypeScript**（现有）
- **React Flow**（画布可视化）
- ** Monaco Editor**（代码节点编辑）
- **Vue Grid Layout**（表单表格）
- **组件库**：DaisyUI + Element Plus

### 后端
- **Django + DRF**（现有）
- **LangGraph**（工作流执行引擎）
- **Pydantic**（DSL 校验）
- **Celery**（向量化异步任务）

### 数据库
- **PostgreSQL**（业务数据）
- **LanceDB/pgvector**（向量存储）
- **Redis**（缓存、消息队列）

### AI/ML
- **OpenAI API / 国产大模型**
- **Embedding 模型**
- **MCP SDK**（技能接入）

## 项目亮点（面试强调）

1. **双模式工作流设计**：画布可视化 + 表单配置，满足不同用户习惯，可随时切换
2. **声明式 DSL**：统一的 JSON 中间格式，支持导入导出，配置即代码
3. **LangGraph 执行引擎**：状态图驱动，支持复杂分支和循环
4. **MCP 技能生态**：标准化协议接入，技能可热插拔
5. **RAG 完整链路**：文档处理 → 向量化 → 检索增强 → 引用溯源
