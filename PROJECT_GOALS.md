# AI Bot 平台 - 项目目标

## 定位

基于现有 **AI 角色聊天系统**，逐步演进为**可配置智能体平台**。不推倒重来，在 `Character` 模型上增加配置层，把硬编码的 LangGraph 能力变成用户可自定义的。

## 现有基础

- ✅ LangGraph 流式对话（SSE）
- ✅ ReAct 工具调用
- ✅ LanceDB 向量检索（RAG）
- ✅ 长期记忆（每 10 轮自动总结）

## 演进路线

| 阶段 | 目标 | 优先级 | 核心改动 |
|------|------|--------|----------|
| **0. 体验优化** | 扩大聊天窗口 + AI 消息 Markdown 渲染 | P0 | `ChatField` 响应式大窗口；消息组件支持代码块、列表等样式 |
| **1. 开场白** | 首次对话自动发送欢迎语 | P0 | `Character` 加 `opening_message` |
| **1.5 历史对话** | 支持多会话管理、清空历史 | P0 | `Friend` 增加会话概念，或支持清空/切换历史 |
| **2. 模型配置** | 不同角色用不同模型/温度/深度思考 | P1 | `Character` 加 `llm_model`、`temperature`、`enable_reasoning` |
| **3. 知识库** | 动态绑定 RAG | P0 | `KnowledgeBase` + `Character.knowledge_bases` |
| **4. 技能广场** | 工具从写死变可配置 | P0 | `Skill` + `Character.skills`，内置智能搜索（联网）等工具 |
| **5. 工作流** | 高级用户可自定义执行路径 | P1 | `Workflow` + `execute_workflow()`，默认仍走 ReAct |
| **6. 多模态** | 用户上传图片列表进行 AI 识图 | P2 | 前端支持图片上传，后端接入多模态模型 |

> **优先级说明**
> - **P0**：面试核心功能，必须完成
> - **P1**：亮点加分项，时间允许就做
> - **P2**：锦上添花，有余力再碰

## 执行计划

### Week 1：可配置智能体 + 基础体验（P0 必须完成）

| 天数 | 任务 | 优先级 | 完成标准 |
|------|------|--------|----------|
| Day 0 | 扩大聊天窗口 + Markdown 渲染 | P0 | 窗口占屏幕 85% 高度；AI 回复支持代码块、列表、引用等样式 |
| Day 1 | 开场白 | P0 | 新建角色可设置，首次聊天自动发送 |
| Day 2 | 历史对话管理 | P0 | 支持清空历史、查看近期会话列表（至少能清历史） |
| Day 3 | 模型选择 + Temperature | P0 | `ChatGraph` 按角色参数动态调用 |
| Day 4 | 知识库模型 + 文档上传 | P0 | 可创建知识库、上传文档 |
| Day 5 | 角色绑定知识库 + RAG 动态化 | P0 | 不同角色检索隔离 |
| Day 6 | 技能广场 + 角色绑定技能 | P0 | 技能可开关，内置"智能搜索"等工具，LangGraph 只加载启用的工具 |
| Weekend | 联调 + 前端页面整合 | P0 | 能完整演示一个带知识库和技能的角色 |

**Week 1 完成标准**：
- 聊天窗口大且 Markdown 渲染正常
- 创建角色时能配置：开场白、模型、温度、知识库、技能
- 支持历史对话管理
- 对话时所有配置项都生效
- 这已经是一个**可演示的智能体平台**

## 最小演示（1 周版）

1. 创建"雅思学习助手"
2. 配置：开场白 + gpt-4 + 温度 0.7 + 绑定口语题库知识库 + 启用智能搜索/时间技能
3. 对话测试：
   - 问天气/时间（观察工具调用）
   - 问知识库内容（观察 RAG 检索）
   - AI 回复带 Markdown 代码块/列表
4. 清空历史后再对话，验证记忆和开场白机制

这就够了，已覆盖：可配置智能体、RAG、技能系统、流式对话、Markdown 渲染、长期记忆。

## 面试话术

> "这个项目最初是固定 AI 角色系统，底层已有 LangGraph 流式对话、ReAct 工具调用、LanceDB 检索和长期记忆。我的演进思路是在 `Character` 上加配置层（模型、温度、知识库、技能），让硬编码能力变用户可配；同时优化基础体验（大窗口、Markdown 渲染、历史对话管理）。对于高级需求，再叠加深度思考开关和可选的工作流编排——默认走 ReAct 降低门槛，高级用户可自定义节点路径。"
